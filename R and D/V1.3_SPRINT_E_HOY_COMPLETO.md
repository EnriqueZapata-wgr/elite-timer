#### 3.A.2 Optimistic update para toggle booleanos (3-5s lag)

Archivo: `src/components/hoy/HoyEditorialSection.tsx:130-156`

Pattern actual: tap → await Supabase x2 → emit → recompile → re-render (3-5s).

Pattern objetivo (igual que `addWater()` en hydration-service.ts:72-127):
1. Calcular `newCompletedValue` en el momento del tap
2. **`setState` local optimista** que palomea la card inmediatamente (sin esperar Supabase)
3. Disparar Supabase + emit async (no await en el handler)
4. Si falla la persistencia → revertir el state local + toast/log de error

Implementación sugerida:

```typescript
const [optimisticOverrides, setOptimisticOverrides] = useState<Record<string, boolean>>({});

const toggleBoolean = (cardKey: string) => {
  if (!userId) return;
  const source = (CARD_TO_ELECTRON[cardKey] ?? cardKey) as ElectronSource;
  if (!source) return;
  if ((VERIFIED_ELECTRON_KEYS as readonly string[]).includes(source)) return;

  // Estado actual (con override optimista si existe)
  const current = optimisticOverrides[source] ?? boolBySource.get(source)?.completed ?? false;
  const next = !current;

  // 1) Optimistic UI palomea/despalomea AHORA
  setOptimisticOverrides((prev) => ({ ...prev, [source]: next }));

  // 2) Persistencia async (fire-and-forget con error handler)
  persistToggle(source, next, current).catch((e) => {
    // Rollback en caso de error
    setOptimisticOverrides((prev) => ({ ...prev, [source]: current }));
    logWarn('[HoyEditorial] toggle failed, reverted', e);
  });
};

const persistToggle = async (source: string, next: boolean, prev: boolean) => {
  const newStates: Record<string, boolean> = {};
  for (const e of day.booleanElectrons ?? []) {
    newStates[e.source] = e.source === source ? next : e.completed;
  }
  const { error } = await supabase.from('daily_electrons').upsert(
    { user_id: userId, date: today, electrons: newStates },
    { onConflict: 'user_id,date' }
  );
  if (error) throw error;
  if (next) await awardBooleanElectron(userId!, source as ElectronSource);
  else await revokeBooleanElectron(userId!, source as ElectronSource);
  DeviceEventEmitter.emit('electrons_changed');
};

// En el render, derivar el state visual con override:
const isCompleted = (source: string) =>
  optimisticOverrides[source] ?? boolBySource.get(source)?.completed ?? false;
```

Limpiar `optimisticOverrides` cuando llegue `electrons_changed` del compiler (para evitar overrides stale).
