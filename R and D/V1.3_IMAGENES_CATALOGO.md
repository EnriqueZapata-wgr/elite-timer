## Guía general de estilo

| Atributo | Especificación |
|---|---|
| **Estilo** | Editorial premium, alto contraste, mood cinematográfico |
| **Color** | Blanco y negro puro (luego gradient overlay agrega color en app) |
| **Resolución mínima** | 1600×900 px (retina 2x para cards full-width) |
| **Aspect ratio** | 16:9 o 4:3 (vertical ok para hero cards) |
| **Composición** | Foco claro, regla de tercios, espacio negativo para texto overlay |
| **Persona** | Diversa, edad 25-50, atletas/profesionales, NO modelos sonriendo cliché |
| **Mood** | Sereno · disciplinado · intenso · introspectivo · vital |
| **A EVITAR** | Stock cliché, sonrisas forzadas, props evidentes, filtros vintage |
| **Formato archivo** | .jpg (cards) o .png (con transparencia si necesario) — comprimir <300KB |

**Fuentes sugeridas (de mejor a más rápida):**
1. **Fotógrafo propio / sesión dedicada** — control total, marca consistente
2. **Midjourney v6+** con prompts editoriales (mejor calidad consistente)
3. **Unsplash Premium** (curado, búsqueda específica)
4. **Pexels / Pixabay** (gratis pero menos curado)
5. **DALL-E 3 / Stable Diffusion** (variable, requiere iteración)

## Plan de acción

### Opción A — Generar con Midjourney (rápido)
Usa los prompts sugeridos. Iterar 4-6 versiones por prompt. Tarda ~1-2h para las 68 si tienes Midjourney v6+.

### Opción B — Curar de Unsplash/Pexels (medio)
Usar los queries sugeridos. Filtrar por estilo editorial B/N. Convertir a B/N si vienen en color (Photoshop / Lightroom). Tarda ~3-4h.

### Opción C — Sesión propia con fotógrafo (slow + caro pero premium)
Coordinar sesión, tomar todas las imágenes con misma luz y estilo. ~1-2 días + costo. Resultado: marca consistente premium.

### Opción D — Híbrido (recomendado)
- **Midjourney** para las 40 de agenda + 6 hoy-extra + 12 YO = 58 (no requieren persona específica)
- **Sesión propia o stock cuidado** para las 2 PILLARS (más críticas, deben gritar marca)
- ~3-4h total

## Decisión Enrique

¿Por cuál vas? Cuando empieces a tener imágenes:
- Colócalas en las carpetas indicadas (asset/images/agenda/, electrons/, etc.)
- Usa los filenames sugeridos para que el código sepa cuáles cargar
- Cuando estén listas, el código hace swap automático del placeholder gradient → imagen real

**Mientras tanto:** las cards funcionan con gradients sólidos placeholder en el sprint actual. NO bloquea desarrollo.

## Notas finales

- Si una categoría no tiene imagen al momento de render → cae a placeholder gradient con icono grande (ya implementado en `EditorialCard`)
- Si quieres mantener TODO en B/N estricto: convertir a grayscale antes de subir
- Si te late ALGO de color sutil (ej. ligero sepia, duotone, monocromo cyan), también ok — el gradient overlay del app le da color encima
