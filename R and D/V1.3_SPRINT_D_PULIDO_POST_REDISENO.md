### 2.3 Fix /solar pantalla cuelga en "Obteniendo datos UV..." (CRÍTICO)

Archivos: `app/solar.tsx` + `src/services/uv-service.ts`

**Causa pinpointed:** sin timeout explícito en `useFocusEffect` → `loadUV()` puede hang indefinidamente si `getCurrentLocation()` (expo-location) no resuelve o si `fetchUVData()` (fetch) se cuelga.

**Cambios:**

1. En `uv-service.ts` `fetchUVData()`: agregar `AbortController` con timeout 10s:
```typescript
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 10000);
try {
  const response = await fetch(url, { signal: controller.signal });
  // ...
} catch (e) {
  if (e.name === 'AbortError') return null; // timeout
  throw e;
} finally {
  clearTimeout(timeoutId);
}
```

2. En `uv-service.ts` `getCurrentLocation()`: si usa `expo-location`, agregar `Promise.race` con timeout 8s:
```typescript
const locationPromise = Location.getCurrentPositionAsync({...});
const timeoutPromise = new Promise((_, reject) =>
  setTimeout(() => reject(new Error('Location timeout')), 8000)
);
const location = await Promise.race([locationPromise, timeoutPromise]);
```

3. En `app/solar.tsx`: si después de timeout `data === null`, mostrar UI de "No pudimos obtener datos UV · reintentar" con botón retry — no quedar en loading forever.
